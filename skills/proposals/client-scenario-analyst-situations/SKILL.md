# Client Scenario Analyst — Situations

Use this skill to generate likely situations, behaviors, and edge cases before writing scenarios.

## Goal

Create a broad situation catalog from the upstream artifacts.
Include normal, exceptional, difficult, and abusive cases when relevant.

## Scope

Typical situations may include:
- best case
- normal case
- rushed case
- inattentive case
- stressed case
- aggressive case
- dishonest case
- misunderstanding
- delay
- no-show
- conflict
- overload
- outage / exception if relevant

## Rules

- Work from the upstream artifacts.
- Keep situations short but concrete.
- Include realistic variety.
- Do not write full stories here.

# Contributing

## Adding a new skill

1. Copy `templates/SKILL.template.md`
2. Create a new folder under `skills/proposals/<skill-name>/`
3. Add `SKILL.md` and any needed `templates/`, `references/`, `assets/`, or `scripts/`
4. Open a PR

## Promoting a skill to canonical

Promote only after:
- at least one real team usage,
- experience notes are understandable,
- the skill is short enough to remain maintainable,
- the team agrees it is reusable.

## Adding an experience note

1. Copy `templates/EXPERIENCE.template.md`
2. Save it under `experiences/<skill-name>/YYYY-MM-DD-short-title.md`
3. Keep it concrete:
   - context
   - what worked
   - what was missing
   - suggested improvement

## Review rules

Reviewers should reject:
- giant monolithic skills,
- vague experience notes,
- speculative changes without real usage,
- direct edits to canonical without review.
